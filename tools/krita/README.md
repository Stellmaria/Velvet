# Velvet Anatomy Logo for Krita

Плагин находится в `tools/krita/velvet_logo` и поддерживает два режима:

1. ручное добавление/удаление логотипа через `Инструменты → Сценарии`;
2. автоматический локальный bridge для Velvet Bot.

## Установка

Соберите ZIP:

```powershell
python tools/krita/package_plugin.py
```

Импортируйте созданный архив в Krita или скопируйте `velvet_logo.desktop` и каталог `velvet_logo` в каталог ресурсов `pykrita`. Затем включите плагин в настройках Python-плагинов и перезапустите Krita.

Krita и бот должны использовать один каталог:

```powershell
$env:VELVET_KRITA_BRIDGE_DIR="E:\VelvetKritaBridge"
```

В `.env` Velvet Bot укажите тот же путь в `KRITA_BRIDGE_DIR`.

Плагин раз в секунду проверяет `requests/*.json`, применяет watermark отдельным векторным слоем, экспортирует новый файл и атомарно записывает ответ в `responses/`. Исходник не перезаписывается. Все пути обязаны находиться внутри bridge-каталога; публичный сетевой порт не используется.

В Krita путь также можно задать через `Инструменты → Сценарии → Velvet Anatomy: настроить bridge-каталог…`.


## Пользовательские логотипы workspace

Bridge schema v2 принимает snapshot логотипа из `assets/`. Поддерживаются очищенный SVG и прозрачный PNG. Файл обязан находиться внутри bridge-каталога; плагин не читает внешние URL и не меняет исходник. Цветовые настройки применяются только к встроенному логотипу Velvet, пользовательский asset сохраняет собственные цвета.
