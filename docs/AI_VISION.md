# Глубокий ИИ-анализ изображений

Velvet может использовать локальную vision-модель для поиска тематических серий у разных персонажей. Анализ не пытается сопоставлять лица. Он выделяет тему, жанр, эпоху, локацию, окружение, предметы, одежду, действие, композицию, свет, палитру и настроение.

Пример: разные персонажи в салуне, пустыне и на ранчо с ковбойскими признаками получают общий смысловой профиль `western` и могут быть предложены как сет «Дикий Запад».

## Почему локально

Изображения отправляются только в локальный сервис на компьютере. Бот продолжает работать без ИИ, если сервис выключен или настройки отсутствуют. Для взрослого художественного архива локальный вариант также не зависит от ограничений облачного vision API.

## Быстрый запуск через Ollama

1. Установить Ollama.
2. Загрузить vision-модель:

```powershell
ollama pull qwen3-vl:8b
```

3. Проверить, что локальный сервис запущен:

```powershell
ollama list
```

Обычно Ollama слушает `http://127.0.0.1:11434`. На Windows приложение может запускать сервис автоматически.

4. Добавить в `.env`:

```dotenv
AI_VISION_ENABLED=true
AI_VISION_PROVIDER=ollama
AI_VISION_BASE_URL=http://127.0.0.1:11434
AI_VISION_MODEL=qwen3-vl:8b
AI_VISION_API_KEY=
AI_VISION_TIMEOUT_SECONDS=180
AI_VISION_MAX_ATTEMPTS=3
```

5. Перезапустить бота.

## OpenAI-совместимый локальный сервер

Можно использовать LM Studio или другой локальный сервер, который поддерживает vision в `/v1/chat/completions`:

```dotenv
AI_VISION_ENABLED=true
AI_VISION_PROVIDER=openai_compatible
AI_VISION_BASE_URL=http://127.0.0.1:1234
AI_VISION_MODEL=имя-загруженной-vision-модели
AI_VISION_API_KEY=
```

## Как работает очередь

- существующие и новые изображения автоматически получают запись в `media_ai_profiles`;
- фоновый процесс анализирует по одному изображению за итерацию;
- исходное изображение уменьшается максимум до 1280 пикселей по длинной стороне перед локальным анализом;
- структурированный JSON-профиль сохраняется в PostgreSQL;
- поиск сетов сравнивает профили независимо от имени и внешности персонажа;
- бот создаёт только предложения, окончательный состав выбирает администратор;
- общий промт по-прежнему хранится один раз на весь подтверждённый сет.

В `/quality` → `Предложения медиасетов` показываются количество готовых профилей, очередь и ошибки.

## Что остаётся отдельно

Поиск точных и почти точных дублей продолжает использовать визуальные fingerprint-признаки. ИИ-анализ не заменяет этот механизм и не должен удалять файлы автоматически.

## Отключение

```dotenv
AI_VISION_ENABLED=false
```

После перезапуска фоновый ИИ-процесс не регистрируется. Архив, поиск дублей и старые безопасные эвристики продолжают работать.
