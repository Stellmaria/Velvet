# Импорт аналитики Telegram

## Основной канал

В Telegram Desktop откройте:

1. `Настройки` → `Продвинутые настройки` → `Экспорт данных из Telegram`.
2. Выберите нужный канал.
3. Формат: `Машиночитаемый JSON`.
4. Для аналитики медиафайлы скачивать не обязательно.
5. После экспорта нужен файл `result.json`.

Отправьте `result.json` в личный чат с `@dominusVelvetbot`, затем ответьте на файл:

```text
/importchannel
```

Статистика доступна командами:

```text
/channelstats
/promptstats
/tagstats
/characterstats
```

## Чат обсуждений

Добавьте бота администратором чата обсуждений и выполните внутри чата:

```text
/trackdiscussion
```

После этого новые сообщения и их правки анализируются автоматически.

Для старой истории экспортируйте сам чат обсуждений в JSON, отправьте его `result.json` боту в личку и ответьте:

```text
/importdiscussion
```

Если ID из экспорта нужно переопределить:

```text
/importdiscussion -1001234567890
```

Отчёт:

```text
/discussionstats
```

## Большие ZIP

Стандартный Telegram Bot API скачивает файлы только до 20 МБ. Если ZIP больше, есть два варианта:

1. Извлечь из ZIP только `result.json` и отправить его боту.
2. Запустить локальный импорт полного ZIP:

```powershell
python scripts/import_telegram_export.py `
  --file "E:\Exports\channel.zip" `
  --kind channel `
  --chat-id -1001234567890
```

Для обсуждения:

```powershell
python scripts/import_telegram_export.py `
  --file "E:\Exports\discussion.zip" `
  --kind discussion `
  --chat-id -1001234567890 `
  --parent-channel-id -1001234567890
```

Скрипт читает только `result.json` внутри ZIP. Фотографии и видео в базу аналитики не копируются.

## Повторный импорт

Импорт идемпотентный:

- сообщения обновляются по паре `chat_id + message_id`;
- один и тот же файл определяется по SHA-256;
- повторная загрузка не создаёт дубли;
- отредактированный экспорт может обновить уже существующие записи.
