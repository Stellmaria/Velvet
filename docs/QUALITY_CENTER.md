# Контроль качества Velvet Archive

Центр открывается владельцем в личном чате с ботом:

```text
/quality
```

## Автоматическое сканирование

После применения миграции `021_media_quality_and_duplicates.sql` все существующие фото и изображения, сохранённые как документы, получают статус `pending`.

Фоновый worker постепенно:

1. скачивает изображение из Telegram;
2. вычисляет SHA-256 исходных байтов;
3. вычисляет pHash полного изображения;
4. вычисляет pHash центральной области;
5. вычисляет dHash и aHash;
6. сравнивает отпечаток с уже обработанными изображениями;
7. сохраняет подозрительные пары в `media_duplicate_candidates`.

Новые изображения автоматически попадают в ту же очередь.

## Порог кандидата

Пара считается кандидатом, если выполняется одно из условий:

- исходные байты совпадают;
- минимальное pHash-расстояние не больше 7;
- pHash не больше 10, dHash не больше 10 и aHash не больше 12.

При сравнении используются также перекрёстные расстояния между полным изображением и центральной областью. Это помогает находить уменьшенные, пережатые и немного обрезанные копии.

## Решения

В карточке пары доступны:

- `Это дубль`;
- `Разные изображения`.

Решение сохраняется в базе. Подтверждение дубля ничего не удаляет. Удаление выполняется отдельно через существующую карточку архива с подтверждением.

## Проверка Telegram-файлов

Worker постепенно вызывает `getFile` для записей архива и сохраняет статус:

- `unknown`;
- `ok`;
- `broken`.

Битые файлы можно вернуть на повторную проверку кнопкой в разделе аудита.

## Разделы аудита

- кандидаты и подтверждённые дубли;
- персонажи без категории;
- персонажи без вселенной;
- персонажи без обязательной истории;
- персонажи без материалов;
- материалы без привязанного поста с промтом;
- недоступные Telegram-файлы;
- ошибки визуального сканирования;
- нераспознанные хэштеги;
- сиротские записи медиа.

Сиротские записи только показываются. Автоматическое удаление отключено.
