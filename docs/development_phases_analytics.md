# План развития Velvet

Документ отражает фактическое состояние `main`, а не первоначальные намерения.

## Фазы 1–6: функциональная основа — завершены

### Фаза 1. Аналитический центр

- единое меню аналитики;
- периоды 7/30 дней и всё время;
- канал, промты, персонажи, хэштеги и обсуждения.

### Фаза 2. Персонажи и классификация

- строгие алиасы;
- очередь нераспознанных тегов;
- ручная классификация публикаций;
- повторная индексация старой истории.

### Фаза 3. Публикации

- проверка поста;
- черновики;
- расписание;
- отправка и повтор после ошибки;
- журнал событий.

### Фаза 4. Качество данных

- perceptual hash;
- поиск дублей;
- контроль обязательных данных;
- битые Telegram `file_id`;
- сиротские связи.

### Фаза 5. Обсуждения и backup

- расширенная аналитика комментариев;
- рейтинги и активность;
- `pg_dump`, ротация и предмиграционная защита.

### Фаза 6. Эксплуатационная устойчивость

- WorkerManager;
- системная диагностика;
- repositories для системных операций;
- безопасное завершение ресурсов.

## Фаза 7: модульная архитектура — основной перенос завершён

### 7A–7F

Завершены composition root и доменные переносы:

- media quality;
- publication;
- characters;
- stories;
- references;
- archive и preview;
- public archive;
- discussions и analytics queries.

### 7G1

Завершены:

- core config;
- access policy;
- Telegram middleware доступа;
- архитектурные regression tests.

Compatibility-фасады ещё присутствуют. Их удаление продолжается отдельными безопасными срезами.

## Фаза 8: Supervisor и управление — завершена

- внешний Supervisor;
- restart, update и rollback;
- логи;
- Codex worktree;
- кнопочный центр;
- подтверждение опасных действий.

## Фаза 9: application use cases владельца — завершена

- профили;
- классификация;
- истории;
- алиасы;
- референсы;
- импорт;
- обсуждения;
- Telegram boundary services.

## Фаза 10: доступ и центр ошибок — завершена

- точная матрица public/editor/owner;
- центр ошибок;
- группировка инцидентов;
- уведомления и подтверждение;
- разделение Supervisor controllers.

## Velvet AI, фазы 1–8 — завершены

1. качество изображения;
2. сравнение с референсом;
3. целостность медиасета;
4. калибровка;
5. единый AI-интерфейс;
6. промт против результата;
7. палитра и композиция;
8. оформление Velvet Anatomy.

## Фаза 11: hardening и deployment — выполняется

### 11A. Документация и версии

- актуальный README;
- changelog;
- статус разработки;
- единая версия Python/PostgreSQL;
- документированная конфигурация Codex и AI.

### 11B. Deployment

- Dockerfile;
- Docker Compose;
- PostgreSQL 16;
- container healthcheck;
- опциональная Ollama;
- production checklist.

### 11C. Проверка восстановления

- restore drill в новую базу;
- контроль миграций и данных;
- запуск в CI при изменениях;
- еженедельная проверка.

### 11D. Архитектурная очистка

Порядок:

1. `analytics_discussion_overrides.py`;
2. `publication_center.py`;
3. `analytics_management.py`;
4. `owner_actions.py`;
5. multi-story compatibility;
6. удаление `presentation/telegram/compat.py`.

Каждый срез выполняется отдельным PR или логически изолированным набором коммитов с PostgreSQL integration tests.

## Вне текущей фазы

- веб-панель;
- автоматическое восстановление рабочей базы;
- автоматическое принятие решений AI без владельца;
- массовая перепись всех compatibility-модулей;
- внешнее облачное хранение backup без шифрования и отдельной политики доступа.
