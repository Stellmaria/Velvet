# Сессия: повторный запуск Qwen и пересканирование дублей

- Дата: 2026-07-20
- ID: `2026-07-20-qwen-duplicate-retry-controls`
- Линия/фаза: P3D stabilization slice
- Статус: `завершено`

## Перед началом

### Цель

Добавить владельцу явные кнопки повторной постановки ошибочных Qwen-заданий и полного пересканирования визуальных дублей.

### Исходный контекст

Qwen quality уже имел bulk-операцию повторной постановки, но она не покрывала semantic-профили и не была доступна из главного аудита. Поиск дублей имел повтор только для ошибочных visual scans, но не имел подтверждаемого полного сброса fingerprints, кандидатов и решений.

### Планируемый объём

- единая операция повторной постановки semantic и quality ошибок Qwen;
- кнопка в Qwen operations и главном аудите;
- автоматический подбор заданий существующими AI workers после постановки;
- подтверждаемый сброс результатов поиска дублей;
- повторная постановка доступных изображений и немедленный запуск media-quality worker;
- регрессионные тесты кнопок и repository contracts.

### Критерии готовности

- ошибочные и skipped Qwen-задания возвращаются в очередь только по явной кнопке;
- оба Qwen workers автоматически подхватывают очередь;
- сброс дублей требует подтверждения;
- fingerprints и кандидаты resettable media удаляются, изображения возвращаются в pending;
- крупные изображения без preview не ставятся в заведомо неисполнимую visual queue;
- CI проходит.

### Риски и ограничения

Полный сброс дублей удаляет прежние решения confirmed/ignored для пересканируемых файлов, поэтому операция должна быть подтверждаемой. Qwen-файлы без доступного thumbnail после ручного повторного запуска снова перейдут в terminal skipped, а не создадут бесконечный цикл.

## После завершения

### Фактически сделано

- существующая кнопка повтора Qwen теперь возвращает в очередь одновременно quality checks и semantic profiles со статусами `error` и `skipped`;
- кнопка `🔄 Ошибки Qwen` добавлена в главный аудит качества;
- периодические workers `ai-quality` и `ai-vision` автоматически подхватывают возвращённые задания;
- в список дублей добавлена кнопка `♻️ Сбросить и проверить заново`;
- перед полным сбросом показывается подтверждение с предупреждением об удалении решений;
- новый domain repository удаляет старые пары и fingerprints для доступных изображений, возвращает их в `pending` и не ставит в очередь файлы больше 20 МБ без preview;
- после подтверждения немедленно запускается первый цикл `media-quality`, остальные файлы обрабатываются периодическим worker;
- добавлены регрессионные тесты repository-операций и callback-кнопок;
- architecture inventory обновлён с 57 до 58 активных routers.

### Миграции и совместимость

SQL-миграции не требуются. Используются существующие таблицы AI checks, semantic profiles, visual fingerprints и duplicate candidates. Callback prefix `quality` и существующие поля payload не меняются.

### Проверки

Добавлены `tests/test_qwen_duplicate_retry_controls.py` и обновлены архитектурные inventory-файлы. Полный GitHub CI запускается в PR.

### PR и commit

PR будет создан из ветки `agent/qwen-duplicate-retry-controls`; итоговый merge commit фиксируется после зелёного CI.

### Незавершённое

Кодовый объём этого среза закрыт. Для файлов Qwen без доступного thumbnail ручной повтор снова завершится terminal `skipped`, что является ожидаемым техническим ограничением cloud Bot API.
