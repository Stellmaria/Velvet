## Перед началом

### Цель

Сделать статус `Просмотрено Стэл` осмысленным: новое фото не должно отмечаться просмотренным при первом открытии. Текущая карточка получает статус только после нажатия стрелки `◀️` или `▶️`, когда Стэл переходит к соседнему материалу.

### Исходный контекст

Callback `open/show` записывал просмотр целевого материала до фактического отображения. Поэтому первое открытие новой карточки сразу превращало `🆕 не просмотрено` в `✅ просмотрено`, хотя Стэл ещё не успела проверить материал.

### Планируемый объём

- сохранить обычные public view metrics для остальных пользователей;
- не записывать owner review при первом `open`;
- передавать в стрелочных callback ID карточки, с которой выполняется переход;
- после успешного перехода записывать просмотр именно покинутой карточки;
- не отмечать новую целевую карточку автоматически;
- сохранить существующие callback prefix, команды, права доступа и PostgreSQL schema;
- добавить regression tests на открытие, переход вперёд и переход назад.

### Критерии готовности

- новая карточка Стэл отображается как `🆕 не просмотрено`;
- простое открытие и закрытие не меняет статус;
- нажатие `▶️` отмечает предыдущую карточку и открывает следующую как новую;
- нажатие `◀️` работает симметрично;
- публичные пользователи продолжают увеличивать обычный счётчик просмотров при открытии;
- полный CI зелёный.

### Риски и ограничения

`public_media_view_stats` одновременно хранит общие просмотры и owner-review marker. Изменение ограничено пользователем `PUBLIC_ARCHIVE_REVIEWER_ID`, чтобы не ломать общую аналитику публичного архива. Owner review записывается после успешной замены карточки, поэтому неудачный Telegram display не считается просмотром.

## После завершения

### Фактически сделано

- первое открытие карточки Стэл больше не вызывает запись owner review;
- стрелки `◀️` и `▶️` передают `media_id` карточки, которую Стэл покидает;
- callback `show` разрешает source `media_id`, одновременно загружая целевой материал по `offset`;
- после успешного показа соседнего материала просмотр записывается для предыдущей карточки;
- новая целевая карточка остаётся `🆕 не просмотрено` до следующего перехода;
- для остальных пользователей обычный view metric продолжает записываться при открытии целевого материала;
- добавлены тесты на source callback, первое открытие, порядок `display → owner review` и обычные public metrics.

### Миграции и совместимость

Миграции не требуются. Таблица `public_media_view_stats`, callback prefix `pub`, команды, права доступа и схема `PublicArchiveCallback` сохранены. В существующее поле `media_id` стрелочного callback передаётся ID текущей карточки; для остальных действий его прежняя проверка на совпадение с целевым материалом сохраняется.
