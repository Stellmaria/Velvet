## Перед началом

### Цель

Разделить одиночное и пакетное сохранение материалов: `/save` должен принимать ровно один поддерживаемый файл и завершать ожидание, а `/save_set` должен оставлять активной пакетную загрузку до явного завершения.

### Исходный контекст

После PR `#309` команда `/save` без ответа на медиа запускала прежнюю пакетную `SaveUploadSession`. Обработчик после каждого файла увеличивал счётчик, продлевал TTL и показывал сообщение `Пакетная загрузка продолжается`. Отдельной команды для пакетного режима не было.

### Планируемый объём

- добавить явный режим `single` или `set` в `SaveUploadSession`;
- сохранить режим `set` по умолчанию для существующих кнопочных сценариев;
- перехватить `/save` без прикреплённого медиа как однофайловое ожидание;
- добавить `/save_set` как пакетный режим;
- закрывать `single` после первого поддерживаемого файла;
- сохранять `set` после каждого файла и показывать прежнюю клавиатуру завершения;
- добавить `/save_set` в workspace access boundary и Telegram command menu;
- добавить регрессионные тесты.

### Критерии готовности

- `/save Имя` без ответа ожидает один файл;
- после первого поддерживаемого файла `/save` больше не перехватывает следующие сообщения;
- `/save_set Имя` принимает несколько файлов до завершения;
- кнопка `Сохранить` продолжает запускать пакетный режим;
- `/save` ответом на конкретное медиа сохраняет его сразу, как раньше;
- editor/owner видит обе команды в Telegram menu;
- viewer не получает изменяющие команды;
- полный CI проходит.

### Риски и ограничения

Telegram-альбом приходит несколькими отдельными updates. В режиме `/save` сохраняется только первое дошедшее поддерживаемое сообщение; остальные не должны автоматически добавляться. Реальный порядок updates проверяется живым smoke test после merge.

### Миграции и совместимость

Миграции БД не требуются. Режим сессии хранится только в оперативной памяти и сбрасывается при перезапуске, как и прежняя пакетная сессия. Старые вызовы `SaveUploadSessions.start()` остаются пакетными благодаря значению `mode="set"` по умолчанию. `/save` ответом на конкретное медиа продолжает использовать прежний немедленный путь сохранения.
