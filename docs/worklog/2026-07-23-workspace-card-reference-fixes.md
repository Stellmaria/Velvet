## Перед началом

### Цель

Исправить создание личного архива после выданного разрешения, убрать устаревшую кнопку промта из карточки персонажа, привести справку карточки материала в соответствие с реальными кнопками и добавить кнопочное управление референсами.

### Исходный контекст

Пользователь успешно видел кнопку создания архива и проходил проверку разрешения, но сообщение с названием перехватывалось глобальным access middleware до создания первого workspace. В карточке персонажа оставалась ссылка на промт, в справке материала перечислялись чужие для этой карточки действия, а добавление референсов всё ещё требовало ручных команд.

### Планируемый объём

- пропустить FSM-состояние ввода названия до проверки существующего workspace;
- убрать ссылку и кнопку промта из личной карточки персонажа;
- объяснить назначение алиаса;
- убрать технический текст с домашней карточки пространства;
- перенести справку по кнопкам материала вниз и оставить в ней только реальные действия карточки;
- добавить кнопки добавления, замены и удаления референсов;
- сохранить workspace boundary, роли и повторные проверки доступа.

### Критерии готовности

- пользователь с активным grant может создать первый архив;
- карточка персонажа не показывает промт и объясняет алиас;
- справка материала находится внизу и описывает только его кнопки;
- владелец или редактор может добавить, заменить и удалить референс кнопками;
- наблюдатель сохраняет read-only доступ;
- тесты и обязательные проектные проверки проходят.

### Риски и ограничения

Замена референса принимает один файл за операцию. Пакетная загрузка остаётся режимом добавления. Живая Telegram-проверка требуется после обновления production.
