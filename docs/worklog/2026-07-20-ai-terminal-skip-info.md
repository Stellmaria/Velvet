# Сессия: терминальные AI-пропуски без предупреждений

- Дата: 2026-07-20
- ID: `2026-07-20-ai-terminal-skip-info`
- Линия/фаза: P3D stabilization hotfix
- Статус: `завершено`
- Ветка: `agent/ai-terminal-skip-info`
- Базовый commit: `871ed286b5f1f2b1f77d2f55620e64f871ad562d`

## Перед началом

### Цель

Перестать создавать WARNING-инциденты для крупных изображений, которые уже корректно переведены в окончательный статус `skipped`, когда Telegram Bot API не позволяет скачать оригинал и не предоставляет доступную миниатюру.

### Исходный контекст

После внедрения устойчивой обработки файлов больше 20 МБ повторный запуск был отключён, а разные `media_id` получили отдельные fingerprints. Однако базовый `MediaAIVisionService.process_once` продолжал логировать любое исключение через `logger.warning`, даже когда ошибка была распознана как постоянная и сохранялась в базе со статусом `skipped`. Поэтому Error Center создавал предупреждения #36, #37 и #38 для файлов 3366, 2464 и 2618, хотя никакого вмешательства или повтора не требовалось.

### Планируемый объём

- классифицировать только точное сообщение терминального oversized-пропуска как INFO;
- не скрывать временные или неизвестные AI-предупреждения;
- применять классификацию только во время итерации worker `ai-vision`;
- добавить regression-тесты уровня логирования и снятия фильтра после итерации;
- не смешивать hotfix с текущим P3D-Backup срезом.

### Критерии готовности

- терминальный `file is too big` с фразой о ненужном повторе имеет уровень INFO;
- Error Center, принимающий записи от WARNING, больше не создаёт такие инциденты;
- предупреждение о недоступности AI-провайдера остаётся WARNING;
- фильтр не остаётся глобально установленным после worker-итерации;
- CI проходит.

### Риски и ограничения

Оригинальный файл без preview всё равно невозможно проанализировать через Telegram Bot API. Исправление меняет только корректность уровня события, не подменяет техническое ограничение и не возвращает такие файлы в очередь.

## После завершения

### Фактически сделано

- добавлен scoped logging-filter для `velvet_bot.ai_vision`;
- точное терминальное сообщение `AI semantic analysis failed` + `file is too big` + `Повтор автоматически не требуется` понижается до INFO;
- worker `ai-vision` запускает одну итерацию через новый wrapper;
- фильтр удаляется в `finally` после каждой итерации;
- остальные AI-предупреждения сохраняют исходный уровень;
- добавлены три regression-теста.

### Миграции и совместимость

Миграции базы данных не требуются. Статусы `media_ai_profiles`, retry-политика, callback-контракты и настройки AI не меняются.

### Проверки

Добавлен `tests/test_ai_vision_terminal_skip_logging.py`. Полный GitHub CI запускается в PR.

### PR и commit

PR создаётся из ветки `agent/ai-terminal-skip-info`; итоговый merge commit фиксируется после зелёного CI.

### Незавершённое

Старые подтверждённые записи #36–#38 останутся в истории Error Center. Они не удаляются автоматически, но после deployment новые записи этого типа возникать не должны.

### Следующий шаг

После merge обновить рабочий бот и продолжить отдельный P3D-Backup срез, не смешивая его commits с данным runtime hotfix.
