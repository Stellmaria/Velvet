## Перед началом

### Цель

Продолжить каноническое ТЗ `docs/requirements/workspace_product.md` и заменить текстовый выбор категории, вселенной и истории личного персонажа на безопасные inline-пикеры Telegram.

### Исходный контекст

Личные персонажи уже были изолированы по workspace и поддерживали собственную taxonomy, но основной экран управления оставался текстовым FSM. Пользователь должен был вводить ключи категорий и вселенных, а также числовые ID историй вручную.

### Планируемый объём

- отдельный picker-router перед текстовым compatibility-контуром;
- пагинация списка персонажей;
- карточка персонажа с собственными label и emoji пространства;
- inline-пикеры категорий, вселенных и историй;
- несколько историй с отметкой основной;
- повторная проверка workspace, роли и module policy на каждом callback;
- защита от подстановки taxonomy ID другого пространства;
- сохранение текстовых команд как запасного интерфейса.

### Критерии готовности

- вход в модуль личных персонажей открывает кнопочный каталог;
- категория и вселенная выбираются без ручного ввода ключа;
- истории назначаются и снимаются кнопками;
- callback не содержит длинных пользовательских ключей;
- чужие ID не проходят серверную проверку;
- системный Velvet Anatomy не меняет текущий интерфейс;
- тесты, type-check, Docker build и notes contract зелёные.

### Риски и ограничения

- создание, переименование, алиасы, prompt URL и Telegram topic пока остаются текстовыми действиями;
- переход к карточке с поздней страницы возвращает в начало списка;
- модуль доступен с минимальной ролью editor, пока отдельный team UI не завершён;
- новая SQL-миграция не требуется.

## После завершения

### Фактически сделано

- добавлен `workspace_character_pickers.py`;
- кнопочный вход перехватывает личный модуль раньше compatibility-router;
- список персонажей разбит на страницы;
- карточка показывает category/universe labels, emoji, медиа, истории, алиасы, prompt и topic;
- category/universe/story picker читает только активную taxonomy текущего workspace;
- смена вселенной очищает старые истории и при `requires_story` сразу открывает выбор истории;
- выбранные истории отмечаются, primary story выделяется отдельно;
- category и universe callback передают числовой row ID, который повторно разрешается внутри workspace;
- прямой callback с ID соседнего пространства отклоняется;
- старый текстовый FSM сохранён для редких операций.

### Миграции и совместимость

Новая миграция не добавлялась. Используются `workspace_categories`, `workspace_universes`, `workspace_stories`, `workspace_character_story_links` и `workspace_character_aliases`.

Системный workspace `1` продолжает использовать прежний раздел персонажей. Публичный каталог и сохранение медиа этим срезом не изменяются.

### Проверки

- запрет чужих category/universe ID;
- выбранную и primary историю;
