## Перед началом

### Цель

Перевести `PublicationRepository` с приватного `Database._require_pool()` на публичный `Database.acquire()` и формально закрепить предметную границу Velvet Archive как отдельного архивного бота для владельца, не связанного с логикой аукционного проекта.

### Архитектурная граница проекта

Velvet Archive обслуживает архив персонажей и авторские рабочие процессы владельца: персонажей, истории, медиа, референсы, публикации, аналитику, AI-проверки и эксплуатацию. В production-пакет нельзя переносить аукционные сущности и сценарии: ставки, лоты, колоды аукциона, валюту, диапазоны ставок, победителей и типы аукционов. Совпадение владельца репозиториев или истории обсуждений не является основанием смешивать домены.

### Исходный контекст

Фазы 18A–18D уже перевели characters, stories, archive, public archive, references и media quality repositories. `PublicationRepository` остаётся следующим записанным срезом и содержит просмотр черновиков, пагинацию, захват публикации, фиксацию успеха или ошибки, очередь расписания и журнал событий.

Поиск production-кода по `auction`, `аукцион` и `bid` не выявил аукционных зависимостей. Это состояние требуется сохранить отдельным архитектурным контрактом, а не полагаться на память очередного человека или агента.

### Планируемый объём

- заменить все обращения `PublicationRepository` к приватному пулу на `self._database.acquire()`;
- сохранить SQL, owner-фильтрацию, пагинацию, статусы и транзакции;
- сохранить `FOR UPDATE SKIP LOCKED` для очереди расписания;
- расширить regression-тест Фазы 18 на publication domain;
- добавить исполняемый тест public acquire и транзакционного event logging;
- добавить контракт предметной изоляции archive production package от auction domain;
- закрепить правило в `AGENTS.md`, project memory и development status;
- определить следующий изолированный repository-срез.

### Критерии готовности

- `PublicationRepository` не содержит `._require_pool()`;
- получение соединений идёт только через `Database.acquire()`;
- owner scope, draft pagination, queue transitions, event records и locked schedule query не меняются;
- production package не содержит импортов или объявлений аукционных доменных сущностей;
- полный workflow tests с PostgreSQL 16, Docker build и project notes contract проходят;
- дневник закрыт точными проверками, PR/commit и следующим шагом.

### Риски и ограничения

- нельзя смешивать этот бот с аукционным проектом даже ради повторного использования похожей команды или модели;
- нельзя превращать почти единоличный owner-инструмент в искусственно сложную многопользовательскую платформу;
- owner_id остаётся границей данных публикаций, но не означает внедрение аукционной ролевой матрицы;
- нельзя менять старые миграции, статусы публикаций и Telegram-delivery;
- несвязанные улучшения публикаций не включаются в этот инфраструктурный срез.
