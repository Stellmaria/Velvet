# Krita SVG и быстрый watermark · hotfix 2026-07-23

## Причина

Задание `77/1` завершилось ошибкой `Krita не смогла импортировать SVG логотипа`.
Бот корректно принял и проверил пользовательский SVG, но установленный Krita plugin
вкладывал его как дочерний SVG внутрь общего документа. `addShapesFromSvg` в Krita
не гарантирует импорт такой вложенной разметки, особенно при `defs`, стилях, масках
и пространствах имён.

## Исправление

- пользовательский SVG остаётся исходным asset пространства;
- plugin `2.1.0` рендерит его через `QSvgRenderer` в прозрачный PNG до передачи в
  `addShapesFromSvg`;
- PNG встраивается тем же способом, который уже используется для растрового логотипа;
- ограничение растра — 4096 px по большей стороне;
- оригинальное изображение задания не перезаписывается.

## Интерфейс

- в модуле watermark кнопка называется `⚡ Быстрый watermark`;
- после нажатия можно отправить следующим сообщением обычное фото или изображение-файл,
  без обязательного ответа на служебное сообщение;
- применяется текущий логотип активного пространства;
- после постановки задания остаются стандартные настройки положения, цвета,
  прозрачности, размера и отступа;
- на карточке материала кнопка быстрого watermark показывается владельцу постоянно;
  при отсутствии логотипа открывается переход в настройку.

## Развёртывание Krita plugin

Обновление кода бота не заменяет уже установленную копию Python plugin в Krita.
После обновления `main` нужно выполнить:

```powershell
python tools\krita\package_plugin.py
```

Затем полностью закрыть Krita, переустановить
`tools\krita\Velvet_Anatomy_Krita_Plugin_bridge.zip`, включить plugin и запустить
Krita снова. В `velvet_logo.desktop` версия отмечена как `2.1.0`.

## Проверка

1. Загрузить проблемный SVG как логотип пространства.
2. Нажать `⚡ Быстрый watermark`.
3. Отправить фото следующим сообщением.
4. Убедиться, что появился preview и работают положение, прозрачность, размер и отступ.
5. Повторить с материалом из карточки личного архива.
