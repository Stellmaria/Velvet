## Перед началом

### Цель

Исправить ошибку пользовательского SVG-логотипа в Krita bridge, вернуть владельцу заметную кнопку быстрого watermark на карточке материала и разрешить отправлять обычное фото в watermark следующим сообщением без обязательного reply.

### Исходный контекст

Задание `77/1` завершилось ошибкой `Krita не смогла импортировать SVG логотипа`. Бот корректно принял и проверил пользовательский SVG, но Krita plugin вкладывал его как дочерний SVG внутрь общего документа. Одновременно быстрый вход был подписан как создание watermark, требовал служебный reply, а карточный маршрут дополнительно зависел от назначения `Watermark-копии`, хотя для формирования отдельного PNG оно не требуется.

### Планируемый объём

- добавить совместимый raster fallback для пользовательских SVG в Python plugin Krita;
- сохранить исходный SVG как asset и не перезаписывать оригинал изображения;
- показать кнопку `⚡ Быстрый watermark` в модуле и карточке материала;
- принимать следующее фото или изображение-документ через FSM;
- применять текущий логотип активного пространства;
- оставить существующие настройки положения, цвета, прозрачности, размера и отступа;
- убрать лишнюю зависимость карточного запуска от storage destination;
- добавить статические regression contracts и инструкцию переустановки plugin.

### Критерии готовности

- проблемный SVG преобразуется plugin в прозрачный PNG перед импортом Krita;
- кнопка быстрого watermark видна владельцу карточки материала;
- при отсутствии логотипа бот предлагает открыть настройки;
- после нажатия в модуле можно прислать фото следующим сообщением;
- создаётся отдельная копия, исходник остаётся неизменным;
- обязательные CI-проверки проходят;
- после переустановки plugin проходит живая Windows/Krita-проверка.

### Риски и ограничения

CI не исполняет реальный Krita Python API. Обновление репозитория и Supervisor не заменяют уже установленную копию plugin в Krita. После merge требуется вручную собрать ZIP, полностью закрыть Krita, переустановить plugin версии `2.1.0` и повторить проблемный SVG на целевой Windows.

## После завершения

### Фактически сделано

- добавлен patch `svg_logo_patch.py`, который рендерит пользовательский SVG через `QSvgRenderer` в прозрачный PNG до вызова `addShapesFromSvg`;
- raster ограничен 4096 px по большей стороне, исходный asset сохраняется;
- patch устанавливается до создания экземпляра `VelvetLogoExtension`;
- в `velvet_logo.desktop` добавлена версия `2.1.0`;
- кнопка модуля переименована в `⚡ Быстрый watermark`;
- добавлено состояние `waiting_source`: фото или изображение-файл можно отправить следующим сообщением;
- добавлен ранний карточный обработчик быстрого watermark до старого destination-gate;
- при отсутствии модуля или логотипа показывается переход в настройки;
- справка карточки актуализирована;
- добавлены статические regression contracts.
