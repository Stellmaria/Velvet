## Перед началом

### Цель

Закрепить в репозитории историю фаз, фактический статус и обязательную практику заметок до и после каждой работы.

### Исходный контекст

В проекте одновременно существовали несколько независимых нумераций:

- ранний рефакторинг;
- основное развитие Velvet Archive;
- Velvet AI/Qwen.

Из-за повторяющихся названий вроде «Фаза 2» часть контекста оставалась только в чатах и PR. Пользователь потребовал вести записи в самом проекте перед началом и после завершения каждой задачи.

### Планируемый объём

- создать единую карту фаз и открытых обязательств;
- создать каталог рабочих сессий и шаблон записи;
- добавить обязательные инструкции для AI-агентов;
- добавить PR-шаблон;
- добавить автоматическую проверку структуры заметок и обязательного обновления worklog;
- обновить текущий статус разработки ссылками на новый контур памяти.

### Критерии готовности

- основные линии фаз явно разделены;
- известные завершённые и открытые задачи записаны;
- любая будущая задача имеет обязательные блоки `Перед началом` и `После завершения`;
- CI отклоняет содержательный PR без новой записи worklog;
- проверка не требует секретов и не зависит от локальной машины;
- текущая сессия закрыта фактическими проверками и PR/commit.

### Риски и ограничения

- старый план раннего рефакторинга восстановлен частично и должен быть помечен историческим;
- нельзя выдавать первоначальные намерения за фактическое состояние `main`;
- слишком строгая CI-проверка не должна блокировать автоматические release/tag события;
- журнал не должен содержать токены, пароли и строки подключения.
