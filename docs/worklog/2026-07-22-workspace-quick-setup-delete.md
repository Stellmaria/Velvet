## Перед началом

### Цель

Сделать первоначальную настройку нового личного пространства реально доступной обычному пользователю, автоматически запросить обязательные Telegram-чаты и каналы после создания архива и добавить безопасное самостоятельное удаление личного пространства.

### Исходный контекст

Мастер первого запуска уже существовал, но его callback-префикс и команды не проходили через общий access middleware. После создания пользователь получал мастер, кнопки которого могли блокироваться. Повторный `/start` не показывал продолжение незавершённой настройки. Привязка назначений требовала вручную выполнять разные команды в чатах и не давала последовательного быстрого сценария ввода `chat_id`. Удаление личного пространства отсутствовало.

### Планируемый объём

- открыть onboarding-команды и callback-префиксы владельцу личного пространства;
- перехватить первый ввод названия раньше старых FSM-handler;
- последовательно запросить `chat_id` или `@username` всех обязательных назначений;
- поддержать необязательный ID темы форума вторым числом;
- проверить административные права, возможность публикации и управление темами;
- показывать продолжение незавершённой настройки через `/start`;
- добавить удаление только личного пространства с отдельным подтверждением;
- перевести блокирующие внешние ключи на каскадное удаление;
- добавить контрактные тесты.

### Критерии готовности

- новый пользователь после ввода имени сразу получает первый запрос быстрой настройки;
- мастер принимает числовой `chat_id` и публичный `@username`;
- форум персонажей проверяется на `is_forum` и право управления темами;
- незавершённый мастер можно продолжить через `/start` и `/workspace_quick_setup`;
- удаление требует отдельного подтверждения и недоступно не-владельцу;
- системное пространство удалить нельзя;
- связанные записи удаляются каскадно;
- новый пользователь после удаления снова может создать архив в пределах действующего grant.

### Риски и ограничения

Telegram не позволяет проверить существование произвольной темы форума только по её ID без сообщения из этой темы. Быстрый мастер сохраняет введённый thread ID после проверки самого чата и прав бота. Закрытые чаты без username подключаются только по числовому ID. Эксплуатационная проверка всё равно нужна на реальных Telegram-группах и каналах.

## После завершения

### Фактически сделано

- добавлен последовательный `workspace_quick_setup` с автоматическим запуском после ввода названия;
- поддержаны форматы `-100...`, `@username` и `-100... THREAD_ID`;
- добавлены проверки администратора, публикации, форума и управления темами;
- незавершённая настройка показывается отдельной кнопкой в `/start`;
- добавлены access extensions для `wob:`, `wqs:`, `wsdel:` и workspace-команд;
- добавлена кнопка удаления личного пространства в `/start`;
- удаление выполняется только владельцем и только после явного подтверждения;
- системное пространство защищено от удаления;
- добавлены сервис удаления, router и тесты жизненного цикла.

### Миграции и совместимость

Добавлена additive migration `911_workspace_self_delete.sql`. Внешние ключи `characters.workspace_id` и `watermark_jobs.workspace_id` заменены с `ON DELETE RESTRICT` на `ON DELETE CASCADE`. Остальные workspace-таблицы уже используют каскадное удаление или `SET NULL`. Существующие пространства не изменяются до явного удаления владельцем.

### Проверки

Добавлены `tests/test_workspace_setup_lifecycle.py`: разбор целей, порядок router, access extensions, наличие кнопок продолжения и удаления, каскадная миграция и обязательное подтверждение. Полный GitHub Actions workflow должен подтвердить форматирование, типы, тесты, миграции и project notes contract.

### PR и commit

Изменения подготовлены в ветке `agent/fix-workspace-onboarding-delete`. Pull request создаётся после финальной проверки diff; итоговый номер PR и head commit фиксируются в GitHub.

### Незавершённое

Нужна живая проверка в Telegram: новый пользователь создаёт архив, вводит ID форумной группы персонажей, темы материалов, референсов и публичного канала, завершает мастер, затем удаляет тестовое пространство и создаёт его повторно. Также следует проверить ввод приватного forum chat ID на реальном боте.
