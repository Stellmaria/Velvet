## Перед началом

### Цель

Добавить двухшаговый режим: команда `/save Артур` без reply включает одноразовое ожидание, а следующее поддерживаемое фото, видео, анимация либо image/video-документ, включая пересланное сообщение, автоматически сохраняется в архив Артура.

### Исходный контекст

Текущий `/save` работает только как reply на уже отправленное медиа. Пользователь хочет сначала выбрать персонажа, а затем прислать или переслать файл. Существующая функция `save_media_from_message` уже выполняет дедупликацию, аудит, сохранение preview и отправку копии в архивную тему, поэтому новый workflow должен переиспользовать её, а не создавать второй путь сохранения.

### Планируемый объём

- добавить короткоживущие one-shot save sessions в application runtime;
- ключевать сессию по chat и caller user;
- валидировать персонажа или быстрый тег до запуска ожидания;
- сохранить существующий reply workflow без изменений;
- перехватывать только поддерживаемое следующее медиа при активной сессии;
- не мешать автоматическому архивированию медиа из назначенных тем;
- добавить `/savecancel`;
- добавить regression-тесты session lifecycle и handler routing;
- пройти полный tests, Docker build и project notes contract.

### Критерии готовности

- `/save Артур` без reply запускает ожидание;
- следующее поддерживаемое медиа сохраняется через общий media save service;
- пересланное сообщение с медиа обрабатывается как обычное входящее медиа;
- сессия одноразовая и истекает автоматически;
- неподдерживаемый документ не завершает сессию;
- `/savecancel` очищает ожидание;
- reply-вариант `/save Артур` продолжает работать немедленно;
- сообщения без активной сессии не перехватываются;
- полный CI зелёный.

### Риски и ограничения

Сессии хранятся в памяти текущего bot process и очищаются при перезапуске, что безопаснее сохранения неожиданного состояния между рестартами. Один запуск принимает один файл. Альбомы из нескольких Telegram-сообщений не входят в этот срез.
