## Перед началом

### Цель

После завершения безопасной консольной команды автоматически присылать инициатору отдельное личное сообщение с итогом, а длинный вывод прикладывать полным UTF-8 текстовым файлом.

### Исходный контекст

После PR #122 карточка команды уже автоматически обновлялась в исходном чате и показывала код возврата, длительность и хвост stdout. Пользователю требовалось отдельное личное уведомление, чтобы итог не потерялся в меню или групповом чате.

### Планируемый объём

1. После terminal-статуса отправлять инициатору отдельное сообщение в ЛС.
2. Сохранять обновление исходной карточки.
3. Для stdout длиннее лимита карточки прикладывать полный `.txt`.
4. Не отправлять вложение для короткого вывода.
5. Не менять allowlist, запуск без shell и маскирование секретов.
6. Ошибка доставки в ЛС не должна влиять на выполнение команды.

### Критерии готовности

- после `success` или `error` приходит отдельное личное сообщение;
- сообщение содержит код возврата, длительность, команду и вывод;
- длинный stdout полностью доступен в `.txt`;
- имя файла безопасно и не содержит разделителей пути;
- короткий stdout не создаёт лишнего документа;
- Telegram API ошибка логируется, но не меняет результат операции;
- unit tests и CI проходят.

### Риски и ограничения

- пользователь должен ранее открыть личный чат с ботом, иначе Telegram может запретить доставку;
- при перезапуске бота во время выполнения фоновый watcher исчезнет, но результат сохранится в истории Supervisor;
- полный stdout уже прошёл существующее маскирование секретов на стороне Supervisor.

## После завершения

### Фактически сделано

- итоговая карточка продолжает обновляться в исходном чате;
- после завершения бот отправляет инициатору отдельное сообщение `📬 Итог команды Supervisor`;
- уведомление содержит status, argv, return code, duration, stdout и error;
- при длинном stdout создаётся `supervisor-console-<operation-id>.txt` с полным очищенным выводом;
- идентификатор операции очищается перед использованием в имени файла;
- короткий stdout остаётся только в сообщении;
- ошибка Telegram при отправке ЛС записывается как warning и не ломает watcher;
- callback подтверждает пользователю, что итог придёт в ЛС.

### Миграции и совместимость

Миграции не требуются. Используется существующий формат operation history и уже очищенный `result.output`.

### Проверки

- добавлены unit-тесты заголовка личного уведомления;
- проверено отсутствие вложения для короткого вывода;
- проверено полное UTF-8 вложение для длинного вывода;
- проверена очистка имени файла;
- требуется GitHub CI после открытия PR.

### PR и commit
