# Сессия: скрытие restricted media в публичном архиве

- Дата: 2026-07-20
- ID: `2026-07-20-public-archive-restricted-media-hotfix`
- Линия/фаза: Velvet Archive, runtime hotfix
- Статус: `завершено`
- Ветка: `agent/hide-restricted-public-media`
- Базовый commit: `24bb50a2b7f94e84c83da3a9554da02be38982fa`

## Перед началом

### Цель

Убрать из публичного архива изображения больше 20 МБ и материалы, которые администратор отметил как +18, а также устранить связанные Telegram-ошибки интерфейса и проверки канала.

### Исходный контекст

Публичный архив показывал карточку +18-материала, после чего пытался проверять подписку на канал с недоступным `ADULT_CHANNEL_ID`. Повторное нажатие административной кнопки могло вызвать `message is not modified`, после чего код повторно отправлял ту же клавиатуру и создавал ложный CRITICAL. Изображения-документы больше 20 МБ также оставались в публичной навигации, хотя cloud Bot API не мог открыть их обычным способом.

### Планируемый объём

- определить единый SQL-контракт публичной видимости media;
- исключить `requires_adult_channel = TRUE` из публичной выдачи;
- исключить изображения больше 20 МБ из публичной выдачи;
- применить контракт к страницам архива, точному открытию, меню и уведомлениям;
- считать `message is not modified` успешным no-op;
- не создавать ERROR при `chat not found` в устаревшей проверке +18-канала;
- сохранить полный доступ менеджера к скрытым материалам.

### Критерии готовности

- обычный пользователь не видит restricted media в меню, навигации и уведомлениях;
- менеджер продолжает видеть и редактировать эти материалы;
- повторное редактирование неизменённой карточки не вызывает исключение;
- `chat not found` возвращает отказ в доступе без ERROR traceback;
- regression tests, Docker build и project notes contract проходят.

### Риски и ограничения

Материалы +18 остаются в базе и административном архиве, меняется только публичная видимость. Для неизвестного `file_size` материал не считается превышающим лимит. Видео и другие не-image документы не скрываются только из-за размера.

## После завершения

### Фактически сделано

- добавлен общий `public_media_visibility_sql` с лимитом `20 * 1024 * 1024` байт;
- public archive page/count, exact media offset, directory filters и notifications используют один visibility contract;
- материалы с `requires_adult_channel = TRUE` полностью исключены из public-only выборок;
- photo/image media больше 20 МБ исключены из public-only выборок;
- оба viewer replacement path считают `message is not modified` штатным no-op;
- `chat not found` в adult membership check логируется как информационное недоступное состояние и возвращает `False`;
- добавлены regression contracts для visibility policy и no-op edits.

### Миграции и совместимость

Миграции БД не требуются. Поля `is_public` и `requires_adult_channel` сохраняются. Менеджерский режим использует `public_only=False`, поэтому скрытые и крупные материалы остаются доступны для управления.

### Проверки
