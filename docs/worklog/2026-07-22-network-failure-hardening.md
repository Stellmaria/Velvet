## Перед началом

### Цель

Убрать HTML/CSS-страницы из ошибок Codex, не открывать инциденты для ожидаемого сбоя `asyncpg.Connection.close()` после уже произошедшего сетевого обрыва и сделать GitHub update устойчивее к кратковременной недоступности сети.

### Исходный контекст

На Windows Codex получил `403 Forbidden` от `chatgpt.com/backend-api/codex/responses`, после чего Supervisor отправил в Telegram полный HTML Cloudflare. Одновременно `asyncpg` создал непросмотренную ошибку из фоновой задачи закрытия уже оборванного соединения. Supervisor Update не смог получить GitHub по HTTPS и оставил сервер на предыдущем commit.

### Критерии готовности

- сообщение Codex 403 короткое и содержит действие `codex --login`;
- HTML/CSS не попадает в Telegram и состояние задачи;
- фильтруется только известная ошибка закрытия соединения после сетевого обрыва;
- остальные asyncio-ошибки передаются стандартному обработчику;
- старые инциденты такого типа закрываются при следующем запуске;
- Git fetch и pull повторяются только для распознанных временных сетевых ошибок;
- грязное рабочее дерево никогда не сбрасывается автоматически;
- полный CI проходит.

## После завершения

### Фактически сделано

- добавлена нормализация ошибок Codex 403 и HTML-ответов;
- установлен точечный asyncio exception guard для `Connection.close()` после сетевого обрыва;
- расширен фильтр Error Incident Center и очистка старых сетевых инцидентов;
- добавлены три попытки для GitHub fetch в dependency sync, обычном Update и self-update;
- добавлены regression tests для 403, asyncio close noise и повторов GitHub fetch.

### Миграции и совместимость

Миграции не требуются. Настройки `.env` и существующие команды Supervisor сохраняются.

### Проверки

Focused tests прошли в temporary workflow. Ожидается финальный полный GitHub Actions CI после удаления служебного workflow.
