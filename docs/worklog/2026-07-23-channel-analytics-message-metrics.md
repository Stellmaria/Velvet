# Channel analytics Bot API metrics hotfix

## Перед началом

### Цель

Устранить повторяющийся сбой захвата публикаций канала, возникающий при обращении к отсутствующим полям `Message.views` и `Message.forward_count` в текущем aiogram Bot API сообщении.

### Исходный контекст

Error Incident Center зарегистрировал инцидент `Failed to capture channel analytics post` с `AttributeError: 'Message' object has no attribute 'views'`. Из-за исключения публикация не доходила до сохранения в `channel_posts`, а одинаковый сбой повторился четырнадцать раз.

### Планируемый объём

- добавить совместимость для Bot API сообщений без счётчиков просмотров и пересылок;
- не подменять отсутствующие метрики нулём;
- сохранить метрики для импортированных или тестовых сообщений, где они присутствуют;
- добавить регрессионные тесты;
- не менять схему базы данных и публичный интерфейс.

### Критерии готовности

- обычный `aiogram.types.Message` без `views` успешно разбирается и сохраняется;
- `view_count` и `forward_count` становятся `NULL`, когда Telegram их не прислал;
- доступные значения продолжают сохраняться;
- обязательные CI-проверки проходят.

### Риски и ограничения

Bot API не гарантирует счётчики просмотров и пересылок в объекте сообщения. Поэтому аналитика не должна представлять отсутствие данных как фактический ноль. Исторические инциденты в лог-чате остаются до ручного подтверждения, но новые повторы после обновления возникать не должны.

## После завершения

### Фактически сделано

- добавлен runtime-адаптер сообщения аналитики;
- отсутствующие `views` и `forward_count` безопасно нормализуются в `None`;
- числовые значения из импортированных сообщений приводятся к `int` и сохраняются;
- адаптер устанавливается до импорта контроллера аналитики;
- добавлены тесты для сообщения без метрик и сообщения с доступными метриками.

### Миграции и совместимость

Миграции не требуются. Столбцы `view_count` и `forward_count` уже допускают `NULL`. Формат существующих записей и команды аналитики не меняются.

### Проверки

Добавлены регрессионные тесты. Финальные номера CI будут внесены в описание pull request после выполнения GitHub Actions.

### PR и commit

Pull request подготавливается из ветки `codex/fix-channel-analytics-message-metrics` в `main`.

### Незавершённое

После merge требуется обновить production через Supervisor и проверить новую публикацию или редактирование публикации в отслеживаемом канале.

### Следующий шаг

Запустить CI, слить исправление в `main`, обновить бота и убедиться, что новый пост сохраняется без инцидента `Failed to capture channel analytics post`.
