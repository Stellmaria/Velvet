# Сессия: открытие крупных архивных изображений файлом для администратора

- Статус: `частично`

- Базовый commit: `8627fafbc317172eca5503edf83dc57d23c62440`

## Перед началом

### Цель

Разрешить администратору открывать сохранённые в архиве документы-изображения размером больше 20 МБ как исходные Telegram-файлы, не пытаясь предварительно скачать их через облачный Bot API для преобразования в фотографию.

### Исходный контекст

Админский архив специально показывает документы-изображения как полноразмерные фотографии. Для этого бот скачивает исходник и создаёт Telegram photo preview. Облачный Bot API ограничивает скачивание файла размером 20 МБ, поэтому крупные PNG/JPEG документы возвращали alert об ошибке, хотя Telegram позволял повторно отправить уже сохранённый `file_id` как документ без скачивания.

### Планируемый объём

- определить крупный документ-изображение по размеру больше `BOT_API_DOWNLOAD_MAX_BYTES`;
- при первом открытии отправлять его через `send_document`;
- при перелистывании строить `InputMediaDocument` вместо `InputMediaPhoto`;
- не запускать preview download для таких файлов;
- сохранить полноразмерный photo preview для изображений до 20 МБ включительно;
- добавить regression-тесты первичного открытия, навигации и порогового поведения;
- не менять публичный архив, хранение, SQL, миграции и права пользователей.

### Критерии готовности

- администратор открывает изображение больше 20 МБ как документ;
- перелистывание на крупный файл не вызывает ошибку preview download;
- файл до 20 МБ продолжает открываться фотографией;
- исходный `telegram_file_id`, подпись, клавиатура и `protect_content` сохраняются;
- unit tests, Docker build и project notes contract проходят.

### Риски и ограничения

Изменение действует только в админском архивном обработчике. Публичная витрина сохраняет прежнюю политику полноразмерных изображений и не получает автоматический доступ к крупным исходным документам. Telegram всё ещё может отклонить испорченный или устаревший `file_id`; такая ошибка продолжит обрабатываться существующим `TelegramBadRequest` boundary.

## После завершения

### Фактически сделано

- добавлен явный режим открытия крупного документа-изображения файлом;
- первичное открытие использует `send_document` с сохранённым Telegram `file_id`;
- навигация использует `InputMediaDocument` и может переключаться на крупный файл;
- preview resolver не вызывается для файлов больше 20 МБ;
- изображения до лимита сохраняют существующий full-quality photo pipeline;
