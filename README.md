# Velvet Archive Bot

Telegram-бот для архива персонажей Velvet. Бот работает на Python 3.14.5, aiogram 3 и PostgreSQL.

## Что реализовано

- создание профиля персонажа;
- привязка персонажа к теме приватной форум-группы;
- автоматический учёт новых фото и видео из назначенной темы;
- сохранение фото и видео через обычную команду и Guest Mode;
- автоматическая отправка сохранённого медиа в тему персонажа;
- защита от повторного создания персонажей и повторного сохранения файлов;
- сохранение исходного имени документа и стабильного имени архива.

## Создание персонажа с темой архива

```text
/create Аид https://t.me/c/3951213065/1398
```

Для существующего персонажа:

```text
/topic Аид https://t.me/c/3951213065/1398
```

Ссылка приватной темы преобразуется в Bot API идентификаторы:

```text
chat_id: -1003951213065
thread_id: 1398
```

Бот проверяет, что группа является форумом и что он состоит в ней администратором. Одна тема может быть назначена только одному персонажу.

После привязки:

- новые фото, видео, анимации и медиа-документы из темы автоматически записываются в архив персонажа;
- медиа, сохранённое из другого чата через `/save` или Guest Mode, отправляется в эту тему по Telegram `file_id`;
- повторный файл не создаётся и не отправляется в тему второй раз.

## Сохранение через Guest Mode

Guest Mode должен быть включён для бота в BotFather.

1. Ответьте на фото или видео в любом поддерживаемом чате.
2. Отправьте:

```text
@username_бота save Аид
```

Telegram передаст боту сообщение-вызов и сообщение, на которое пользователь ответил. Бот сохранит медиа в PostgreSQL и отправит его копию в назначенную тему Аида.

## Обычное сохранение

В чате, где бот уже состоит:

```text
/save Аид
```

Команда должна быть отправлена ответом на фото или видео.

## Важное ограничение старой истории

Telegram Bot API не предоставляет метод чтения полной истории группы или темы. Даже администратор получает только новые входящие обновления; необработанные обновления Telegram хранит не дольше 24 часов.

Поэтому бот автоматически архивирует новые сообщения после запуска и привязки темы. Для старых материалов нужен один из вариантов:

- переслать старые фото и видео повторно в назначенную тему;
- сохранить их ответом через `/save` или Guest Mode;
- использовать отдельный MTProto-импортёр от пользовательского аккаунта. Такой импортёр не входит в обычный Bot API и должен настраиваться отдельно.

## Требования

- Python 3.14.5;
- PostgreSQL;
- токен бота от `@BotFather`.

## Установка на Windows

```powershell
git clone https://github.com/Stellmaria/Velvet.git
cd Velvet

py -3.14 -m venv .venv314
.\.venv314\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt

Copy-Item .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=токен_от_BotFather
DATABASE_URL=postgresql://velvet:пароль@localhost:5432/velvet
```

PostgreSQL можно установить как обычную службу Windows. Docker не требуется.

## Запуск

```powershell
python main.py
```

При старте бот подключается к PostgreSQL, применяет миграции и подписывается на `message` и `guest_message`.

## Проверка

```powershell
python -m unittest discover -s tests -v
```

Интеграционные тесты PostgreSQL запускаются только с отдельной тестовой базой:

```powershell
$env:TEST_DATABASE_URL="postgresql://velvet:пароль@localhost:5432/velvet_test"
python -m unittest discover -s tests -v
```

Никогда не задавайте рабочую базу в `TEST_DATABASE_URL`: интеграционный тест очищает таблицы.
