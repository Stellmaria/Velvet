# Velvet Archive Bot

Telegram-бот для приватного архива персонажей Velvet. Бот работает на Python 3.14.5, aiogram 3 и PostgreSQL.

## Что реализовано

- создание профиля персонажа;
- привязка персонажа к теме приватной форум-группы;
- автоматический учёт новых фото и видео из назначенной темы;
- сохранение фото и видео через `/save`, обычное упоминание и Guest Mode;
- автоматическая отправка сохранённого медиа в тему персонажа;
- просмотр архива по кнопкам с полноразмерным предпросмотром изображений;
- удаление медиа из архива с подтверждением;
- защита от повторного создания персонажей и повторного сохранения файлов;
- owner-only доступ по Telegram ID или username;
- Telegram-аудит добавлений, удалений, отправок в ветки и ошибок.

## Ограничение доступа

По умолчанию бот разрешает команды, Guest Mode, inline-запросы и callback-кнопки только пользователю `@va_stellmaria`.

```env
ALLOWED_USERNAMES=va_stellmaria
ALLOWED_USER_IDS=
```

Числовой Telegram ID надёжнее username, потому что username можно изменить.

## Лог-чат

```env
LOG_CHAT_ID=-5367533184
```

Бот должен быть добавлен в этот чат и иметь право отправлять сообщения. Туда приходят:

- запуск и остановка;
- добавление медиа;
- автоматическое принятие медиа из ветки;
- отправка копии в ветку персонажа;
- удаление медиа;
- ошибки обработчиков и Telegram API.

## Создание персонажа с темой архива

```text
/create Аид https://t.me/c/3951213065/1398
```

Для существующего персонажа:

```text
/topic Аид https://t.me/c/3951213065/1398
```

## Сохранение медиа

В чате, где бот уже состоит, ответьте на фото или видео:

```text
/save Аид
```

Также работает обычное упоминание:

```text
@dominusVelvetbot save Аид
```

В любом поддерживаемом чате через Guest Mode используется тот же вызов ответом на медиа:

```text
@dominusVelvetbot save Аид
```

Guest Mode включается в BotFather.

## Inline-список по `@`

Чтобы бот появлялся в списке, похожем на `@gif`, включите Inline Mode:

```text
@BotFather → /setinline → @dominusVelvetbot
```

Inline Mode используется только для подсказки. Он не получает сообщение, на которое пользователь ответил. Для реального сохранения нужно выйти из inline-режима и отправить обычное упоминание `@dominusVelvetbot save Аид` ответом на медиа.

## Просмотр и удаление

```text
/character Аид
```

Кнопки:

- `Открыть архив` — показывает последнее медиа;
- `◀️` и `▶️` — листают архив;
- `🗑 Удалить` — открывает подтверждение;
- `Ветка` — открывает связанную тему;
- `Закрыть` — удаляет сообщение просмотра.

Изображения, сохранённые как PNG/JPG-документы, временно скачиваются в память и показываются как крупная фотография. В ветке персонажа исходный файл остаётся документом без подписи.

Удаление убирает связь из PostgreSQL. Если файл больше не связан ни с одним персонажем, удаляется и строка `media_files`. Бот также пытается удалить свою копию из ветки, но Telegram ограничивает удаление старых сообщений.

## Важное ограничение старой истории

Telegram Bot API не предоставляет чтение полной старой истории темы. Автоматически учитываются только новые сообщения, которые бот получает после запуска и привязки темы.

## Установка на Windows

```powershell
git clone https://github.com/Stellmaria/Velvet.git
cd Velvet

py -3.14 -m venv .venv314
.\.venv314\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt

Copy-Item .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=токен_от_BotFather
DATABASE_URL=postgresql://velvet:пароль@localhost:5432/velvet
ALLOWED_USERNAMES=va_stellmaria
ALLOWED_USER_IDS=
LOG_CHAT_ID=-5367533184
```

PostgreSQL можно установить как обычную службу Windows. Docker не требуется.

## Запуск

```powershell
python main.py
```

## Проверка

```powershell
python -m unittest discover -s tests -v
```

Интеграционные тесты PostgreSQL запускаются только с отдельной тестовой базой:

```powershell
$env:TEST_DATABASE_URL="postgresql://velvet:пароль@localhost:5432/velvet_test"
python -m unittest discover -s tests -v
```

Никогда не задавайте рабочую базу в `TEST_DATABASE_URL`: интеграционный тест очищает таблицы.
