# Velvet Archive Bot

Telegram-бот для архива персонажей канала Velvet. Бот работает в личных сообщениях, группах и супергруппах, где он добавлен.

## Реализовано

- запуск через long polling;
- команда `/start`;
- создание профилей персонажей в SQLite;
- защита от повторного создания одного имени;
- список персонажей и просмотр отдельного профиля;
- подготовленная схема архива файлов с защитой от дубликатов.

## Требования

- Python 3.13;
- токен Telegram-бота от `@BotFather`.

## Установка на Windows

```powershell
git clone https://github.com/Stellmaria/Velvet.git
cd Velvet

py -3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt

Copy-Item .env.example .env
```

Откройте `.env` и замените значение `BOT_TOKEN` на токен бота:

```env
BOT_TOKEN=1234567890:telegram_token_here
DATABASE_PATH=data/velvet.db
```

## Запуск

```powershell
python main.py
```

База создаётся автоматически в `data/velvet.db` при первом запуске.

## Команды персонажей

```text
/create Каин
```

Создаёт новый профиль персонажа. Также поддерживается вариант `/crete Каин`, чтобы случайная опечатка не превращалась в отдельный жизненный кризис.

```text
/characters
```

Показывает список созданных персонажей.

```text
/character Каин
```

Показывает профиль персонажа и количество связанных с ним изображений.

Имена сравниваются без учёта регистра и лишних пробелов. Поэтому `Каин`, `КАИН` и `  Каин  ` считаются одним персонажем.

## Архив изображений

Схема базы уже подготовлена для следующего этапа:

- оригинальное имя файла сохраняется отдельно;
- Telegram `file_unique_id` используется для определения одинакового файла;
- один физический файл хранится один раз;
- один файл при необходимости можно связать с несколькими персонажами без создания копий.

Команда сохранения изображений `/save Имя` будет добавлена следующим этапом.
