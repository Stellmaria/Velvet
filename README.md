# Velvet Archive Bot

Telegram-бот для приватного архива персонажей Velvet. Бот работает на Python 3.14.5, aiogram 3 и PostgreSQL.

## Что реализовано

- создание профиля персонажа;
- привязка персонажа к теме приватной форум-группы;
- автоматический учёт новых фото и видео из назначенной темы;
- сохранение фото и видео через `/save`, обычное упоминание и Guest Mode;
- автоматическая отправка сохранённого медиа в тему персонажа;
- просмотр архива по кнопкам с полноразмерным предпросмотром изображений;
- блюр 18+ для фото, видео, анимаций и превью больших изображений;
- удаление медиа из архива с подтверждением;
- категории, вселенные и истории персонажей;
- расширенная панель архива для владельцев и редактора `8179531132`;
- сбор аналитики новых и отредактированных постов основного канала;
- статистика промтов, хэштегов, персонажей, медиа, ссылок и структуры постов;
- защита от повторного создания персонажей и повторного сохранения файлов;
- owner-only доступ по Telegram ID или username;
- Telegram-аудит добавлений, удалений, отправок в ветки и ошибок.

## Ограничение доступа

По умолчанию бот разрешает служебные команды только пользователю `@va_stellmaria`.

```env
ALLOWED_USERNAMES=va_stellmaria
ALLOWED_USER_IDS=
```

Числовой Telegram ID надёжнее username, потому что username можно изменить. Все владельцы из `ALLOWED_USER_IDS` и `ALLOWED_USERNAMES` получают в открытом архиве те же кнопки, что и редактор `8179531132`:

- скачать оригинал;
- включить или снять блюр 18+;
- удалить материал;
- изменить пол / состав;
- изменить вселенную;
- изменить историю.

## Лог-чат

```env
LOG_CHAT_ID=-5367533184
```

Бот должен быть добавлен в этот чат и иметь право отправлять сообщения. Туда приходят запуск, остановка, добавление и удаление медиа, ошибки и состояние подключения каналов аналитики.

## Аналитика основного канала

Основной канал настроен по умолчанию:

```env
ANALYTICS_CHANNEL_IDS=-1003802812639
```

Бота нужно добавить администратором канала. После этого он получает обновления `channel_post` и `edited_channel_post` и сохраняет:

- публикации и отдельные сообщения альбомов;
- дату публикации и редактирования;
- тип медиа;
- наличие спойлера 18+;
- текст и его длину;
- хэштеги;
- ссылки и домены;
- зафиксированные просмотры и пересылки;
- признаки блоков `ВАЖНО`, `СТРОГО`, Negative, технического блока и HEX-палитры;
- совпадения хэштегов с карточками персонажей;
- категорию, вселенную и историю найденного персонажа.

Команды владельца:

```text
/channelstats
/promptstats
/tagstats
/tagstats #Аид
/characterstats
```

`/channelstats` показывает общую статистику, типы контента, частые домены, топ хэштегов и персонажей. `/promptstats` анализирует структуру распознанных промтов. `/tagstats` считает использование тегов. `/characterstats` показывает, какие персонажи чаще задействованы в канале.

Палитры вида `#A17C6B` не учитываются как хэштеги.

## Создание персонажа с темой архива

```text
/create Аид https://t.me/c/3951213065/1398
```

Для существующего персонажа:

```text
/topic Аид https://t.me/c/3951213065/1398
```

## Сохранение медиа

В чате, где бот уже состоит, ответьте на фото или видео:

```text
/save Аид
```

Также работает обычное упоминание:

```text
@dominusVelvetbot save Аид
```

В любом поддерживаемом чате через Guest Mode используется тот же вызов ответом на медиа.

Guest Mode включается в BotFather.

## Inline-список по `@`

Чтобы бот появлялся в списке, похожем на `@gif`, включите Inline Mode:

```text
@BotFather → /setinline → @dominusVelvetbot
```

Inline Mode используется только для подсказки. Он не получает сообщение, на которое пользователь ответил. Для реального сохранения нужно выйти из inline-режима и отправить обычное упоминание ответом на медиа.

## Просмотр и удаление

```text
/character Аид
```

Кнопки служебного архива позволяют листать, блюрить, привязывать промт, удалять материал, открывать ветку и закрывать просмотр.

Изображения, сохранённые как PNG/JPG-документы, временно скачиваются в память и показываются как крупная JPEG-картинка. Исходный файл остаётся без изменений и доступен владельцу для скачивания.

Удаление убирает связь из PostgreSQL. Если файл больше не связан ни с одним персонажем, удаляется и строка `media_files`. Бот также пытается удалить свою копию из ветки, но Telegram ограничивает удаление старых сообщений.

## Ограничение старой истории Telegram

Telegram Bot API не предоставляет боту полную старую историю тем и каналов. Автоматическая аналитика начинается с момента, когда бот добавлен администратором и запущен. Для исторической статистики нужен импорт экспорта Telegram Desktop.

## Установка на Windows

```powershell
git clone https://github.com/Stellmaria/Velvet.git
cd Velvet

py -3.14 -m venv .venv314
.\.venv314\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt

Copy-Item .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=токен_от_BotFather
DATABASE_URL=postgresql://velvet:пароль@localhost:5432/velvet
ALLOWED_USERNAMES=va_stellmaria
ALLOWED_USER_IDS=
LOG_CHAT_ID=-5367533184
ANALYTICS_CHANNEL_IDS=-1003802812639
```

PostgreSQL можно установить как обычную службу Windows. Docker не требуется.

## Запуск

```powershell
python main.py
```

## Проверка

```powershell
python -m unittest discover -s tests -v
```

Интеграционные тесты PostgreSQL запускаются только с отдельной тестовой базой:

```powershell
$env:TEST_DATABASE_URL="postgresql://velvet:пароль@localhost:5432/velvet_test"
python -m unittest discover -s tests -v
```

Никогда не задавайте рабочую базу в `TEST_DATABASE_URL`: интеграционный тест очищает таблицы.
